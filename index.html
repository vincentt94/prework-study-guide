HTML

  • HTML stands for hypertext markup language which gives content structure
  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • < h1 > represents the head element
  • < p > represents paragraph or a block of texts element
  • < ul > is urnordered list, < ol > is ordered list, and < li > is the list
  • < br > creates a line of empty space or a line break
  • < a > or anchor creates links to the webpage follwoed by href=

CSS

  • CSS stands for cascading style sheets which give content style
  • inline CSS styles an element directly in the HTML file
  • internal style sheet embeds an entire filesheet directly in the HTML file
  • external style sheet allows developers to keep all their CSS rules in a separate file making design changes easier
  • a margin indicates how much space we want around the outside of an element
  • a padding indicates how much space we want around the content inside an element
  • a selector definse the element or attributes to which the rules, or declarations, will apply
  • declarations contain two important components: the CSS property we want to apply & the value of the property

Git

  • Git is a version control system that lets you manage and keep track of each version of your project
  • Git is valuable because it allows for multiple people to make separate changes to a program at the same time without getting in each others way
  • Github is the web interface version for Git
  • Git Bash is the application for windowws that provides an emulation layer for a Git command experience
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • " git pull origin main " pulls the remote main branch (from Github) into the local main branch (onto the laptop)
  • " git push origin xyz " pushes whatever changes made from our local enviroment (laptop) from the feature branch titled xyz onto Github

Javascript

  • often abbreviated as JS, Javascript is a high level dynamic programing language that allowws users to interact with websites
  • Variables are often used in JS to contain & store values
  • different types of variables include: string (sequence of text), numbers, Boolean (a true/false statement), arrays ( a structure that allows you to store multiple values in a single reference), & object ( this can be anything and all of the previous are examples of an object)
  • const xyz = represents a variable (xyz) that CANNOT be reassigned
  • let xyz = represents a variable (xyz) that CAN be reassigned often used within a FOR loop
  • FOR loops usually have 3 parts: the starting point, the condition, & the final statement
  • console.log (xyz) = outputs a message to the web console by adding an argument, it will output whatever is within the parentheses
  • === is a strict equality operator which checks to see if two values are equal & returns a boolean results (true/false); true if value are equal, false if values are not equal
  • within arrays, length is used to determine the amount within an array
  • example) const topics = ['HTML', 'CSS', 'Git', 'Javascript']; for ( let x = 0; x < topics.length; x++ ) {console.log(topics[x]); }
  • in the above example, topics is the variable with an array of data including HTML, CSS, Git, & Javascript; the topics.length segment represents the length of the array topics

VS code to Github

  • Updates made from VS code must be transfered correctly to Github
  • git status ---> shows us what branch we are currently in
  • git checkout -b abc ----> creates and moves us to a feature branch named abc
  • git add -A ---> commits all files into the working feature branch
  • git commit -m "abcdefghijk" ---> -m is used to associate a message with our commmit, the actual commmit is within the parentheses (abcdefghijk)
  • git pull origin main ---> pulls in base branch from VS
  • git push origin abc ---> pushes changes made from feature branch abc into Github repo
  • Now we must create a Pull Request on Github, to merge feature branch abc (compare) into the main branch (base)
  • Add a title & short description to the PR
  • Now you can merge the new feature branch into the main branch (this is usually checked by a peer prior to merging)
