HTML

  • HTML stands for hypertext markup language which gives content structure
  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • < h1 > represents the head element
  • < p > represents paragraph or a block of texts element
  • < ul > is urnordered list, < ol > is ordered list, and < li > is the list
  • < br > creates a line of empty space or a line break
  • < a > or anchor creates links to the webpage follwoed by href=

CSS

  • CSS stands for cascading style sheets which give content style
  • inline CSS styles an element directly in the HTML file
  • internal style sheet embeds an entire filesheet directly in the HTML file
  • external style sheet allows developers to keep all their CSS rules in a separate file making design changes easier
  • a margin indicates how much space we want around the outside of an element
  • a padding indicates how much space we want around the content inside an element
  • a selector definse the element or attributes to which the rules, or declarations, will apply
  • declarations contain two important components: the CSS property we want to apply & the value of the property

Git

  • Git is a version control system that lets you manage and keep track of each version of your project
  • Git is valuable because it allows for multiple people to make separate changes to a program at the same time without getting in each others way
  • Github is the web interface version for Git
  • Git Bash is the application for windowws that provides an emulation layer for a Git command experience
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
